Overview

System Overview

NoHack is a paired-device messaging system. The NoHack phone is air-gapped — permanently offline, holding all keys, contacts, and plaintext. The Relay phone is online, connecting to Telegram as a stateless, encrypted pipe. The two phones connect via a USB-C cable. No setup required — plug in and it works.

Zero Trust Relay
The security model assumes the Relay is fully compromised. It has internet access, runs Telegram, and is exposed to all remote attack vectors. Even with full Relay access, an attacker cannot read, forge, or tamper with messages.

Encryption

Encryption

All messages are encrypted using NaCl box (Curve25519 + XSalsa20-Poly1305) via the tweetnacl library. Each message uses a fresh ephemeral keypair for forward secrecy.

Key Generation

On first launch, NoHack generates a Curve25519 keypair. The public key is stored in AsyncStorage. The secret key is stored in the Android Keychain (with an AsyncStorage fallback).

Encrypted Payload Format

base64-encoded packed payload
ephemeralPubKey 32 bytes
nonce 24 bytes
ciphertext variable

Forward Secrecy

Each outgoing message generates a fresh ephemeral Curve25519 keypair. The ephemeral secret key is used once for encryption and then discarded. Even if the sender's long-term key is compromised, past messages remain secure.

Decryption

NoHack extracts the ephemeral public key and nonce from the packed payload, then performs nacl.box.open with its own secret key. If decryption fails (wrong recipient), it throws a DecryptionError and silently drops the message.

What the Relay Sees
Only the base64-encoded encrypted blob. The Relay cannot decrypt messages (no secret key), forge messages (no sender's secret key), or read any content at any stage.

USB Communication

USB Communication

The two phones communicate via Android Open Accessory (AOA) protocol over USB-C. The Relay acts as USB Host and initiates the AOA handshake. NoHack acts as USB Accessory and responds.

Transport Protocol

Newline-delimited JSON over USB bulk transfer (16KB chunks).

CommandDirectionPurpose
decryptRelay → NoHackDeliver encrypted message
introductionRelay → NoHackDeliver contact card
pingRelay → NoHackCheck if NoHack is alive
ackNoHack → RelayAcknowledge ping / read receipts
encryptedNoHack → RelaySend encrypted message out

Data Flow

Relay (Host) NoHack (Accessory) UsbHostModule.kt ──────────────────► UsbConnectionModule.kt bulkTransfer() inputStream.read() (16KB chunks) (16KB buffer) │ │ ▼ ▼ RelayUsbService.ts UsbService.ts write() handleData() newline-delimited extractLines() JSON parseTransportLine() │ │ ▼ ▼ RelayMainScreen.tsx ChatListScreen.tsx message router message router

USB Hardening

USB Device Filter
Only accepts manufacturer="NoHack" model="NoHack Relay". No wildcard fallback — unknown USB devices are ignored entirely.
Buffer Cap
1 MB maximum buffer size. If exceeded, the entire buffer is dropped and the connection is reset. Prevents memory exhaustion attacks.
Command Whitelist
Only 3 commands accepted: decrypt, introduction, ping. All other commands are silently dropped.
No Code Execution
No eval(), no dynamic require(), no shell access. USB data is parsed as JSON only — never executed.

Routing Architecture

NoHack stores all contact and routing data. The Relay is completely stateless — zero contacts, zero routing tables. Relays can be swapped anytime without losing any data.

Security

Security Model

The Relay phone is assumed fully compromisable. It has internet access, runs Telegram, and faces all remote attack vectors. The security model is designed so that even total Relay compromise cannot breach message confidentiality.

What a Compromised Relay CANNOT Do

Read Messages
NaCl box encryption. Relay has no secret key — only sees opaque base64 blobs.
Forge Messages
Inner payload includes senderPublicKey. Forging requires the sender's secret key.
Execute Code on NoHack
USB protocol only accepts 3 JSON commands. No eval, no shell, no dynamic code loading.
Access NoHack Filesystem
Android Accessory mode provides app-level I/O only. No filesystem, no ADB, no MTP/PTP.
Install Malware on NoHack
Accessory USB mode cannot push APKs, access ADB, or modify the system.
Brute-force Decryption
Ephemeral keys per message. Curve25519 provides ~128-bit security level.

What a Compromised Relay CAN Do (Mitigated)

Inject Fake Introduction
New contacts show verification warning: "Verify this person out-of-band before sharing secrets."
Redirect Replies
relayTelegramId changes are detected and logged with console.warn.
Impersonate Relay via USB
USB filter requires exact manufacturer and model match.
Memory Exhaustion
USB buffer hard-capped at 1 MB. Excess data is dropped entirely.
